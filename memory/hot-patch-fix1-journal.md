## Functions Changed
1. **`hasReplayableThinkingSignature(block)`** — now delegates to `isValidThinkingSignature()` instead of bare `typeof === "string" && trim().length > 0`
2. **`stripInvalidThinkingSignatures(messages, options)`** — no longer unconditionally preserves the latest assistant message's thinking blocks. All messages (including latest) run through the same `hasReplayableThinkingSignature` validation.
3. **New: `isValidThinkingSignature(signature)`** — validates base64 format + decoded byte length 32-512

## Behavior Change
Previously: Latest assistant message thinking blocks were always preserved verbatim, regardless of signature validity. A corrupted thinking block with an invalid Anthropic ed25519 signature would be replayed, causing Anthropic to reject with "thinking block...invalid signature."

Now: Thinking blocks with invalid signatures (empty, non-base64, wrong byte length <32 or >512) are stripped from ALL assistant messages, including the latest. Blocks with valid signatures are preserved.

- Old `stripInvalidThinkingSignatures`: had `if (i === latestAssistantIndex) { out.push(msg); continue; }` skip
- New `stripInvalidThinkingSignatures`: all messages pass through the same validation loop
